## Архитектура

```
┌─────────────────────────────────────────────┐
│               Render.com                     │
│  ┌─────────────────────────────────────────┐ │
│  │          server.js (Node.js)            │ │
│  │                                         │ │
│  │  ┌──────────┐  ┌───────────────────┐    │ │
│  │  │ Telegram  │  │  Express API      │    │ │
│  │  │ Bot       │  │  /api/tracks      │    │ │
│  │  │ (Telegraf)│  │  /api/tracks/stream│    │ │
│  │  └──────────┘  └───────────────────┘    │ │
│  │                                         │ │
│  │  ┌──────────┐  ┌───────────────────┐    │ │
│  │  │ Frontend  │  │  SQLite DB        │    │ │
│  │  │ HTML/JS   │  │  (dj_bot.sqlite)  │    │ │
│  │  │ CSS       │  │                   │    │ │
│  │  └──────────┘  └───────────────────┘    │ │
│  └─────────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
         │                    │
         ▼                    ▼
   Telegram API       Пользователь
   (Bot + Files)      (Telegram Mini App)
```

**Ключевой принцип:** Фронтенд и API обслуживаются одним сервером. Это устраняет проблемы с CORS и позволяет API-вызовам работать по тому же домену.

## Первоначальная настройка

### 1. Создание бота в Telegram

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте `/newbot` (или используйте существующего бота)
3. Задайте имя и username
4. Скопируйте **BOT_TOKEN** (формат: `1234567890:AABBccDDeeFF...`)
5. Сохраните токен — он понадобится на следующем шаге

### 2. Подготовка репозитория

Убедитесь, что репозиторий `axis-alex/dj-mini-app` на GitHub содержит файлы:
- `server.js` — сервер + бот
- `database.js` — SQLite схема
- `index.html`, `app.js`, `styles.css` — фронтенд
- `package.json` — зависимости
- `render.yaml` — конфигурация Render

## Деплой на Render.com

### Шаг 1: Создание аккаунта

1. Перейдите на [render.com](https://render.com)
2. Зарегистрируйтесь через **GitHub** (это упростит привязку репозитория)

### Шаг 2: Создание Web Service

1. В Dashboard нажмите **New** → **Web Service**
2. Выберите **Build and deploy from a Git repository** → **Next**
3. Подключите репозиторий `axis-alex/dj-mini-app`
4. Заполните настройки:

| Поле | Значение |
|------|----------|
| **Name** | `katz-studio` (или любое другое) |
| **Region** | `Frankfurt (EU Central)` (ближайший) |
| **Branch** | `main` |
| **Runtime** | `Node` |
| **Build Command** | `npm install` |
| **Start Command** | `node server.js` |
| **Plan** | `Free` |

### Шаг 3: Переменные окружения

В разделе **Environment Variables** добавьте:

| Key | Value | Описание |
|-----|-------|----------|
| `BOT_TOKEN` | `1234567890:AABBcc...` | Токен от @BotFather |
| `WEBAPP_URL` | `https://katz-studio.onrender.com` | URL вашего сервиса на Render |
| `NODE_VERSION` | `18.20.7` | Версия Node.js |

> ⚠️ **WEBAPP_URL** должен совпадать с URL, который Render присвоит вашему сервису. Формат: `https://<service-name>.onrender.com`

### Шаг 4: Деплой

1. Нажмите **Create Web Service**
2. Render начнёт сборку (~2-3 минуты)
3. В логах вы должны увидеть:
   ```
   🚀 Server running on port 10000
   Connected to the SQLite database.
   🤖 Telegram bot started
   ```
4. После деплоя ваш сервис доступен по URL: `https://katz-studio.onrender.com`

### Шаг 5: Проверка

1. Откройте `https://katz-studio.onrender.com/api/health` — должен вернуть JSON:
   ```json
   {"status":"ok","version":"4.0.0","uptime":12.345}
   ```
2. Откройте `https://katz-studio.onrender.com` — должен отобразиться DJ-миксер

## Настройка бота в Telegram

### Установка Web App URL

1. Откройте [@BotFather](https://t.me/BotFather)
2. Отправьте `/setmenubutton`
3. Выберите вашего бота
4. Отправьте URL: `https://katz-studio.onrender.com`
5. Отправьте текст кнопки: `🎧 Mixer`

### Проверка бота

1. Откройте вашего бота в Telegram
2. Отправьте `/start` — бот должен ответить с кнопкой "Открыть Mixer"
3. Отправьте аудиофайл — бот должен ответить "✅ Трек сохранён!"
4. Нажмите кнопку "🎧 Открыть Mixer" — должен открыться миксер
5. В миксере нажмите "💬 Telegram" — должен появиться список треков

## Обслуживание и мониторинг

### Логи

В Dashboard Render → ваш сервис → **Logs** — здесь видны все запросы и ошибки в реальном времени.

### Health Check

Render автоматически перезапускает сервис при падении. Вы также можете проверить вручную:
```
curl https://katz-studio.onrender.com/api/health
```

### Бесплатный план — ограничения

| Ограничение | Описание |
|-------------|----------|
| **Спящий режим** | Сервис засыпает через ~15 минут бездействия. Первый запрос после сна занимает ~30-60 секунд |
| **SQLite** | База данных хранится на диске и **сбрасывается при каждом деплое**. Треки, добавленные через бота, будут потеряны |
| **Время сборки** | 750 часов/месяц бесплатно |

> 💡 **Рекомендация по SQLite**: На бесплатном плане Render диск эфемерный. Треки сохраняются между запросами, но пропадают при новом деплое. Для долгосрочного хранения рассмотрите подключение внешней БД (Turso, Supabase, или Render PostgreSQL).

### Прогрев (Keep Alive)

Чтобы сервис не засыпал, можно использовать бесплатный пинг-сервис:

1. Зайдите на [cron-job.org](https://cron-job.org)
2. Создайте задачу:
   - URL: `https://katz-studio.onrender.com/api/health`
   - Интервал: каждые 14 минут
3. Это предотвратит засыпание сервиса

## Обновление приложения

### Автоматический деплой

Render автоматически деплоит при каждом пуше в `main`:

```bash
# Внесите изменения в код
git add .
git commit -m "update: описание изменений"
git push origin main
# Render автоматически начнёт пересборку (~2-3 мин)
```

### Ручной деплой

В Dashboard Render → ваш сервис → **Manual Deploy** → **Deploy latest commit**

### Обновление токена бота

1. Получите новый токен: @BotFather → `/revoke` → `/token`
2. В Render Dashboard → **Environment** → обновите `BOT_TOKEN`
3. Нажмите **Manual Deploy** для применения

## Устранение неполадок

### "Ошибка загрузки треков" в приложении

| Причина | Решение |
|---------|---------|
| Приложение открыто через GitHub Pages | Откройте через кнопку бота (URL должен указывать на Render) |
| Сервис Render заснул | Подождите ~30 сек, обновите страницу |
| BOT_TOKEN невалидный | Обновите токен в Environment Variables |
| WEBAPP_URL не совпадает | Убедитесь, что WEBAPP_URL = URL сервиса на Render |

### Бот не отвечает

1. Проверьте логи в Render Dashboard
2. Убедитесь, что в логах есть `🤖 Telegram bot started`
3. Если ошибка `401: Unauthorized` — токен невалиден, обновите его

### SQLite ошибки

Если в логах `Error opening database`:
- Убедитесь, что у приложения есть права на запись в директорию
- На Render это обычно работает автоматически

### Треки пропали после деплоя

Это нормально на бесплатном плане Render — диск эфемерный. Пользователь может заново отправить треки боту.

## Локальная разработка

### Запуск

```bash
cd dj-mini-app-main

# Создайте .env
echo "BOT_TOKEN=ваш_токен" > .env
echo "PORT=3000" >> .env
echo "WEBAPP_URL=http://localhost:3000" >> .env

# Установка зависимостей
npm install

# Запуск
npm start
```

### Тестирование API

```bash
# Health check
curl http://localhost:3000/api/health

# Открыть миксер
open http://localhost:3000
```

> ⚠️ Telegram Mini App `initData` будет пустым при открытии в обычном браузере. Для тестирования загрузки треков из Telegram нужен реальный Telegram-клиент.
